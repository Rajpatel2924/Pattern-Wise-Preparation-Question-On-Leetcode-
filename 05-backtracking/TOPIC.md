# Backtracking

Build solutions incrementally and abandon (“backtrack”) when a partial solution cannot lead to a valid final answer.

Common scenarios:
- Subsets/combinations/permutations
- N-Queens / Sudoku-like constraint filling
- Word search in grids

Checklist:
- Define state + choices at each level
- Add/remove choices cleanly (undo steps)
- Prune invalid branches early
